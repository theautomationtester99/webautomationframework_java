Web Automation Framework - Help Page

About This Project

This Web Automation Framework is designed to automate browser interactions, execute test scripts, and generate detailed reports. It supports various keywords for performing specific actions, such as opening a browser, interacting with elements, and validating test results.

Command-Line Arguments

  • --start: Start the execution of test scripts.
  • --start-parallel: Start parallel execution of test scripts.
  • --version: Display the version information of the framework.
  • --encrypt-file: Encrypt a specified file.
  • --output-file: Specify the output file name (used with --encrypt-file).
  • --help-html: Open this help page in the default browser.

Keywords and Their Usage

The following keywords are supported by the framework. Each keyword performs a specific action during test execution.

1. open_browser

Opens a browser (e.g., Chrome or Edge) for test execution.

Put the keyword open_browser in the Keyword column and specify the browser name (e.g., chrome) in the Input3 column.

Example:
Keyword Input1 Input2 Input3
open_browser chrome

2. enter_url

Enters a URL into the browser's address bar.

Put the keyword enter_url in the Keyword column and give the input URL in the Input3 column.

Example:
Keyword Input1 Input2 Input3
enter_url https://example.com

3. click

Clicks on an element identified by a locator from the object repository.

Put the keyword click in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
click login_proceed_button_xpath

4. type

Types text into an input field identified by a locator from the object repository.

Put the keyword type in the Keyword column, specify the locator in the Input2 column, and the text to type in the Input3 column.

Example:
Keyword Input1 Input2 Input3
type login_uname_textbox_xpath test_user

5. drag_drop

Performs a drag-and-drop operation between two elements identified by locators from the object repository.

Put the keyword drag_drop in the Keyword column, specify the source locator in the Input1 column, and the target locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
drag_drop htatras1_dragable_css htatras_droppable_css

6. wait_for_seconds

Pauses the execution for a specified number of seconds.

Put the keyword wait_for_seconds in the Keyword column and specify the number of seconds in the Input3 column.

Example:
Keyword Input1 Input2 Input3
wait_for_seconds 5

7. verify_displayed_text

Verifies that a specific text is displayed on the page.

Put the keyword verify_displayed_text in the Keyword column, specify the locator in the Input2 column, and the expected text in the Input3 column.

Example:
Keyword Input1 Input2 Input3
verify_displayed_text demoqa.elements.heading Welcome, User!

8. switch_to_iframe

Switches the context to an iframe identified by a locator from the object repository.

Put the keyword switch_to_iframe in the Keyword column and specify the iframe locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
switch_to_iframe checkbox_iframe_css

9. switch_to_default_content

Switches the context back to the default content from an iframe.

Put the keyword switch_to_default_content in the Keyword column. No additional inputs are required.

Example:
switch_to_default_content (no additional inputs required).

10. select_file

Selects a file to upload using a file input element identified by a locator from the object repository.

Put the keyword select_file in the Keyword column, specify the locator in the Input2 column, and the file path in the Input3 column.

Example:
Keyword Input1 Input2 Input3
select_file file_input_xpath C:\path\to\file.txt

11. check_element_enabled

Checks if an element identified by a locator is enabled.

Put the keyword check_element_enabled in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
check_element_enabled submit_button_xpath

12. check_element_disabled

Checks if an element identified by a locator is disabled.

Put the keyword check_element_disabled in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
check_element_disabled submit_button_xpath

13. check_element_displayed

Checks if an element identified by a locator is displayed on the page.

Put the keyword check_element_displayed in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
check_element_displayed header_xpath

14. check_radio_chk_selected

Checks if a radio button or checkbox identified by a locator is selected.

Put the keyword check_radio_chk_selected in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
check_radio_chk_selected radio_button_xpath

15. check_radio_chk_not_selected

Checks if a radio button or checkbox identified by a locator is not selected.

Put the keyword check_radio_chk_not_selected in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
check_radio_chk_not_selected radio_button_xpath

16. hover_mouse

Moves the mouse pointer over an element identified by a locator.

Put the keyword hover_mouse in the Keyword column and specify the locator in the Input2 column.

Example:
Keyword Input1 Input2 Input3
hover_mouse menu_item_xpath

Configuration Parameters (start.properties)

The start.properties file contains various configuration parameters that control the behavior of the framework. Below is a detailed description of each parameter:

1. [Misc]

Parameter Purpose Values Default
delete_test_results_images_recordings_folders_before_start Deletes the folders test_results, images, and recordings before starting execution. yes, no yes
screenshot_strategy Determines when screenshots are taken during execution. never, always, on-error always
highlight_elements Highlights elements during execution to visually indicate interactions. yes, no no

2. [SGrid]

Parameter Purpose Values Default
run_in_selenium_grid Enables or disables execution in a Selenium Grid environment. yes, no no
grid_url Specifies the URL of the Selenium Grid hub. Example: http://localhost:31820/wd/hub N/A

3. [Appium]

Parameter Purpose Values Default
run_in_appium_grid Enables or disables execution in an Appium Grid environment. yes, no no
appium_url Specifies the URL of the Appium server. Example: http://192.168.1.8:4723/wd/hub N/A

4. [Logging]

Parameter Purpose Values Default
LogLevel Sets the logging level for the framework. debug, info, warning, error, critical debug

5. [Browser_Settings]

Parameter Purpose Values Default
InPrivate Runs the browser in private/incognito mode. yes, no yes
Headless Runs the browser in headless mode (without a visible UI). yes, no yes

6. [Parallel]

Parameter Purpose Values Default
NoThreads Specifies the number of threads to use for parallel execution. Any positive integer 4
